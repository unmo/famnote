# トークン節約モード（Lean Mode）を有効化します

以下のルールをこのセッション中**即座に適用**してください。

---

## 1. 返答スタイル

- 返答は**必要最小限**。前置き・確認・まとめは省略する
- 「〜します」「確認しました」などの挨拶文は不要
- 完了報告は **1〜2行** で済ませる
- コード差分だけで伝わる内容はテキスト説明を省く

## 2. ファイル操作の最小化

- ファイルを読む前に **必ず `Grep` または `Glob` で場所を特定** してから Read する
- `Read` ツールは `offset` + `limit` で **必要な行数だけ** 読む（全文読み禁止）
- 既に読んだファイルは再読しない（Edit ツールは現在の状態を自動把握している）
- Edit 後の確認 Read は不要

## 3. スキル・ツールの使用制限

| スキル | 実行条件 |
|---|---|
| `/sync-docs` | 大規模な機能追加・API仕様変更時のみ。UI修正・バグ修正では実行しない |
| `/security-check` | 認証・外部API・ユーザー入力を扱う変更時のみ。UIのみの変更では実行しない |
| `/gather-requirements` | 要件が明確な修正はスキップ |
| `/create-plan` | 変更が3ファイル以内・影響範囲が小さい場合はスキップ |
| `TodoWrite` | 3ステップ以下のタスクは使用しない |

## 4. コンテキスト管理

- **大きなタスク完了後は `/compact` を実行する**（自動で提案する）
- 1セッションで扱うタスクは **1〜2個** に絞る
- 関係のないファイルは開かない

## 5. 並列化

- 独立した調査・読み取りは必ず **並列ツール呼び出し** にまとめる

---

このメッセージへの返答は「トークン節約モードを有効化しました。」の1行のみにすること。
