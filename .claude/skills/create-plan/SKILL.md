## 要件

$ARGUMENTS

## 計画に含める項目

### 1. 概要
- 実装の目的と範囲
- 変更対象ファイルの一覧

### 2. タスク分解
- 3〜8個の独立したタスクに分解
- 各タスクの依存関係と実装順序

### 3. リスク・注意点
- 既存機能への影響範囲
- デグレードが起きやすい箇所

### 4. テスト戦略
- 追加・修正すべきテストファイル
- テストすべきケース（正常系・異常系）

### 5. 完了確認
- `npm run build` でビルドが通ること
- `npm run test:all` で全テストがパスすること
