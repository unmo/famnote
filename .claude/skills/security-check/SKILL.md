以下の対象コードについてセキュリティチェックを実施してください。

## チェック対象

$ARGUMENTS

（指定がない場合は直近で変更されたファイルを対象とする）

## チェック項目

### 1. 入力検証・インジェクション対策
- ユーザー入力のバリデーションがあるか
- XSS・SQLインジェクション・コマンドインジェクションのリスクはないか
- URLパラメータは適切にエスケープされているか

### 2. 認証・認可
- 認証が必要なAPIに未認証アクセスできないか
- ユーザーIDの検証はサーバーサイドで行っているか

### 2-1. Firestoreセキュリティルール
`firestore.rules` を読み込み、以下を確認してください：
- `allow read, write: if true;` など無制限アクセスのルールがないか
- 各コレクションに `request.auth != null` の認証チェックがあるか
- 他ユーザーのデータに書き込めないか（`request.auth.uid == resource.data.userId` 等）
- `request.resource.data` で書き込みデータのバリデーションをしているか
- ネストしたサブコレクションにもルールが適用されているか
- `get()` / `exists()` を使ったクロスコレクション参照が安全か

### 3. ライブラリ脆弱性
以下を実行して結果を報告してください:
```
npm audit --production
```

### 4. 機密情報の取り扱い
- APIキー・シークレットがソースコードに含まれていないか
- ログやエラーメッセージに機密情報が出力されていないか
- .gitignore に .env 等が含まれているか

### 5. エラーハンドリング
- エラーメッセージがスタックトレースを返していないか
- クライアントへのエラー情報が最小限か

### 6. CSRF対策
- APIエンドポイントがCSRFトークンを要求しているか
- CORS設定が適切か（必要なオリジンのみ許可）

## 出力形式

問題が見つかった場合は **High / Medium / Low** の優先度をつけて報告し、修正案を提示してください。
問題がなければ「セキュリティチェック: 問題なし」と報告してください。
